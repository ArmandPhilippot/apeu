This is an MDX page! Its purpose is to test the rendering of the associates HTML/Markdown elements. You can access it using `/mdx`.

## A heading

Mapping headings to our Astro component should be supported, as well as:

- lists and list items
- [link](/)
- `inline code`
- <kbd>keystrokes</kbd>
- keystrokes combination such as <kbd><kbd>Ctrl</kbd> + <kbd>C</kbd></kbd>
- onscreen input <kbd><samp>options</samp></kbd>
- <samp><kbd>echoed input</kbd></samp> from a program
- <samp>output</samp> from a program
- <q>inline quotations</q>

Ordered lists should also be supported:

1. item 1
2. item 2
3. item 3

You can also use blockquote:

> For you quotations

## Images

### With Markdown syntax

You should also be able to use images with Markdown syntax:

![an image](../../../src/assets/favicon.png)

### With HTML syntax

Or using an HTML tag:

<img alt="an HTML img" src="../../../src/assets/favicon.png" />

### Remote images

<img alt="a remote img" src="https://picsum.photos/640/480" />

### Clickable images

You can also use clickable images with both syntax:

[![a clickable image](../../../src/assets/favicon.png)](../../../src/assets/favicon.png)

The link will not be updated if it does not target the img source:

<a href="https://astro.build">
  <img alt="an HTML img" src="../../../src/assets/favicon.png" />
</a>

## Complex structures

Sometimes you may need more complex structures like:

### Asides

You can use Markdown directives in your content by specifying a type after the opening of the directive to create asides:

:::warning
This is an example of an aside using the `warning` type.
:::

You can provide a title using `[The title]`:

:::idea[This is the title]
This is an example of an aside using the `idea` type and a title.
:::
